# Web Dev for People Who Don’t Want to Be Confused Anymore

This document provides step-by-step instructions and explanations to accompany a beginner video series. It is designed to make web development concepts clear and actionable for absolute beginners.

---

## 5‑Minute Video → “What a Website Actually Is”

**Goal:** Build a mental model for domains, DNS, hosting, frontend, backend.

### Key Concepts

- **Domain:** The human-friendly address for your site (e.g., example.com).
- **DNS:** The system that connects your domain to your website’s server.
- **Hosting:** The service that stores and serves your website files to the world.
- **Frontend:** The part of the website users see (HTML, CSS, JS).
- **Backend:** The part that runs on the server (APIs, databases, logic).

**Takeaway:**
> “Ohhhh… now I get what a domain, DNS, hosting, frontend, backend actually mean.”

## 10‑Minute Video → “Your First Deployment”

**Goal:** Experience the magic of deploying a real website.

### Step-by-Step

1. **Write Code:**
   - Create an `index.html` file with basic content.
2. **Create a GitHub Repo:**
   - Sign up/log in to GitHub.
   - Create a new repository and upload your `index.html`.
3. **Deploy to Cloudflare Pages:**
   - Sign up/log in to Cloudflare Pages.
   - Connect your GitHub repo.
   - Deploy your site (Cloudflare builds and hosts it for free).
4. **See Your Live Website:**
   - Visit the provided URL to see your site online.

**Takeaway:**
> “I can actually build something real.”

## 15‑Minute Video → “Static vs Dynamic + Modern Web Architecture”

**Goal:** Understand how modern web apps work and the difference between static and dynamic sites.

### Key Concepts

- **Static Hosting:**
  - Serves fixed files (HTML, CSS, JS). Fast and simple.
- **Dynamic Hosting:**
  - Runs code on the server (APIs, databases, user logins).
- **APIs:**
  - Interfaces for your site to talk to other services or databases.
- **Databases:**
  - Store and retrieve data for your site.
- **Serverless:**
  - Run backend code without managing servers (e.g., serverless functions).
- **Modern Hosting:**
  - Platforms like Cloudflare, Vercel, Netlify automate builds, deploys, and scaling.

**Takeaway:**
> “I finally understand how modern apps work.”
